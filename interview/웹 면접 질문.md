# 웹 면접 질문

## 1. HTTP 통신 방식

1. HTTP
   - www상에서 서버와 클라이언트가 정보를 주고 받을 수 있는 프로토콜로, HTML 문서를 주고 받는데 사용된다
   - TCP 방식: client-server 중 한곳이 연결을 끊을때까지 연결 유지
   - http 방식: client가 server에서 html을 다운받는 것을 끝내면 연결을 끊어버린 stateless
   - 특징
     - stateless
       - 클라이언트와 서버가 연결을 성공한 후, 클라이언트의 요청에 대해 서버가 응답을 하고 나면 연결을 끊어버린다
       - 장점
         - 통신간의 연결 상태 처리나 정보의 저장을 할 필요가 없음
         - 즉, 서버의 디자인이 간단해짐
       - 단점
         - 연결을 끊어버리기 때문에 서버에서 클라이언트의 현재 상태를 알 수 없음.
         - 로그인 유지가 안된다는 말
     - 요청과 응답
       - 요청
         - 웹 브라우저에서 웹 서버로 보내는 요청.
         - 서버에게 데이터를 전송할 때 Request 객체에 담아 전달하게 된다
       - 응답
         - 웹 서버에서 웹 브라우저에게 응답해줌
         - 클라이언트로부터 서버에게 요청이 일어나면 서버는 브라우저에 전달할 데이터를 response객체를 통해 전달한다

## 2. Cookie와 Session

http통신은 stateless이기 때문에 정보를 저장하기 위해 쿠키와 세션을 이용한다

- 차이점
  - 쿠키는 클라이언트쪽에 저장
  - 세션은 서버쪽에 저장
  - 보안 면에서는 세션이 더 우수하다
  - 처리 속도에 있어서는 쿠키가 더 우수하다

1. Cookie
   - 사용자의 디스크나 웹 브라우저 메모리에 저장
   - key, value형태로 text타입으로 저장한다
   - 크기에 한계가 있다
   - 프로세스
     1. 브라우저에서 웹 페이지 접속
     2. 웹 서버에서 쿠키 생성
     3. 생성한 쿠키에 데이터를 담아 요청에 응답할 때 클라이언트에게 전송
     4. 클라이언트가 보관하다가 서버에 재요청할 때 쿠키를 함께 전송
     5. 클라이언트와 서버가 로그인 정보가 유지되어있는 것처럼 사용
2. Session
   - 웹 서비스를 위한 사용자의 정보를 서버측에 저장
   - 서버 측에서 객체 형태로 저장된다
   - 프로세스
     1. 클라이언트가 서버에 접속 시 세션 ID를 발급
     2. 클라이언트는 쿠키를 이용해 세션 ID를 저장해서 가지고 있음
     3. 클라이언트가 서버에 재요청시 쿠키를 이용하여 세션ID값을 서버에 전달
